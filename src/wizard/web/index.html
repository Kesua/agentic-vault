Connect Slack

Step 1: Create a Slack App

  1. Open Slack App Management
  2. Click "Create New App" and choose "From an app manifest"
  3. Pick your workspace
  4. Paste this YAML manifest:
display_information:
  name: Vault Slack Read
features:
  bot_user:
    display_name: Vault Slack Read
    always_online: false
oauth_config:
  scopes:
    bot:
      - channels:history
      - channels:read
      - groups:history
      - groups:read
      - im:history
      - im:read
      - mpim:history
      - mpim:read
      - users:read
      - users:read.email
      - chat:write
      - chat:write.public
      - im:write
      - mpim:write
      - users:read
settings:
  interactivity:
    is_enabled: false
  org_deploy_enabled: false
  socket_mode_enabled: false
  token_rotation_enabled: true

Step 2: Install to Workspace

In the app settings, go to "OAuth & Permissions" and click "Install to Workspace".

Step 3: Paste Bot Token

After installing, copy the Bot User OAuth token from the OAuth page.

Install Playwright Browser

This optional setup installs a local Codex browser plugin plus Chromium for real-browser tasks.

What it enables

  1. Open and inspect real web pages in Chromium
  2. Take screenshots and read DOM content
  3. Capture network responses from browser-gated sites

Supported on Windows and macOS. The installer creates a local home plugin outside the vault and registers it for Codex.

Local AI Setup

This optional setup helps you run a private AI model on your computer instead of sending requests to a cloud provider.

If you choose it, the wizard will check your hardware, suggest a suitable local model, connect OpenCode to LM Studio, and verify the local server.

1. Scan This Machine

The wizard checks your hardware first so it can recommend whether Gemma 4 E4B is a good fit or whether a smaller model is the safer starting point.

2. OpenCode

OpenCode is the CLI this wizard connects to your local model host.

On Windows, OpenCode works best through WSL, but the native Windows install still works as a fallback.

3. Install LM Studio and Download a Model

  1. Download and install LM Studio
  2. Open LM Studio and search for a runnable Gemma model
  3. Use the recommendation above to decide whether to start with Gemma 4 E4B or a smaller model
  4. For capable machines, start with a 4-bit quantized Gemma 4 E4B variant

4. Start the Local Server

In LM Studio, start the OpenAI-compatible local server. This wizard expects the base URL below.

5. Write OpenCode Config

The wizard can save the LM Studio provider config either for this machine or just for this project.

LM Studio accepts a placeholder value here. The default is fine.

6. Final Verification

This checks that OpenCode is installed, the config exists, LM Studio is reachable, and at least one local model is loaded.
